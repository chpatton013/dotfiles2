# This request — before you start

- More than a step or two? Drive it with the `execute-plan` skill: smallest
  working slices first, riskiest first, ship (commit) each increment.
- Any real design, interface, or refactor decision → load the `software-design`
  skill and follow it *before* writing the code, not from memory.

Using a skill means reading its `SKILL.md` and doing what it says. Not optional.
